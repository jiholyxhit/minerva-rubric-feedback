# Available rubrics

This index lists the rubrics currently supported by the skill. Each rubric has a reference file in this directory with its definition, strong examples, and evaluation criteria.

| Rubric | File | One-line summary |
|---|---|---|
| `#audience` | `audience.md` | Tailor work to the situation and perspective of the receiver. |
| `#constraints` | `constraints.md` | Identify and apply constraint satisfaction to solve problems. |
| `#levelsofanalysis` | `levelsofanalysis.md` | Analyze a complex system by integrating explanations across multiple levels and their interactions. |
| `#purpose` | `purpose.md` | Articulate a mission statement that coherently connects values, goals, and operating procedures. |
| `#composition` | `composition.md` | Communicate with a clear, precise, parsimonious, and relevant style. |
| `#connotation` | `connotation.md` | Understand and deliberately use connotations, tone, and style in word choice. |
| `#thesis` | `thesis.md` | Formulate a well-defined, arguable, and specific thesis. |

## Adding a new rubric

Copy `audience.md` as a template and fill in each section:

1. **Rubric definition** — one or two sentences, ideally the exact wording the school uses.
2. **Strong example — body application** — a concrete passage that demonstrates the rubric well.
3. **Strong example — contextualization footnote** — the footnote that would accompany that passage.
4. **Why this pair works** — spell out *why* the example is strong. This is what lets the skill recognize the pattern in new work.
5. **Evaluation criteria** — split into body and footnote, with bullet-level criteria.
6. **Common weaknesses** — a table with weakness / what-it-looks-like / stronger-version.

Then add a row to the table above.
